Task Model

Structure, lifecycle, and fields for Forge task files.

Required fields

Field Description
id Unique identifier: TASK-YYYYMMDD-NNN
title Short descriptive title
status todo | doing | paused | blocked | done
priority A (urgent) | B (normal) | C (low)
project Repository/project name
tags Array of string tags for filtering
context Execution context: dev, ops, etc.
description What needs to be done and why
acceptance Checklist of acceptance criteria
verification How to verify completion (commands, checks)
paths Files/directories this task will touch
created_at ISO date of creation
updated_at ISO date of last update

Status lifecycle

todo → doing → done
doing → paused → doing
doing → blocked → doing
pausedWork intentionally paused; not a blocker. Resume with /resume-task.
blockedCannot proceed due to external dependency. Optionally set blocker_type: paused.

Task file example

---
id: TASK-20260222-001
title: "Add rate limiting to the API"
status: todo
priority: A
project: my-app
tags: [api, security, rate-limit]
context: dev
created_at: 2026-02-22
updated_at: 2026-02-22
iteration_id: 12
---

## Description

Add per-IP rate limiting to the public API endpoints using a token bucket
algorithm. Prevent abuse without blocking legitimate high-volume clients.

## Acceptance

- [ ] Rate limiting active on all /api/v1/* routes
- [ ] Limit: 100 req/min per IP, configurable via environment variable
- [ ] 429 response includes Retry-After header
- [ ] Tests cover limit boundary and reset behavior

## Verification

```
npm test -- --filter=rate-limit
curl -I http://localhost:3000/api/v1/status
```

## Paths

- src/middleware/rate-limit.ts (new)
- src/server.ts (apply middleware)

## Updates

2026-02-22: Created (iteration 12, new-feature workflow).

Sprint and iteration fields

Optional fields added in v0.4:

sprint_idSprint this task belongs to, e.g. "sprint-2026-02"
iteration_idForge iteration that created this task, e.g. 24

Task actions

/task-create

Create a new task from description

/task-update

Update status or fields on a task

/task-list

List tasks with optional filters

/backlog-task

Capture an idea to the backlog
