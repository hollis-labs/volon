Artifact Classes

Five distinct artifact classes used in Volon, each with a defined intent, audience, and location.

The word "docs" is overloaded. Use these class names and intent: frontmatter instead.

Five artifact classes

Class Intent value Audience Location
System Docs system_doc Humans docs/
Project Docs project_doc Humans docs/ or project root
Knowledge Artifacts knowledge_artifact Humans + Agents artifacts/knowledge/
PCC pcc Agents only .volon/pcc/
Bootstrap bootstrap Humans + Agents .volon/bootstrap.md

Key rules

PCC is not docs. PCC is agent-only context. Updating PCC is not "updating documentation."
Bootstrap is not PCC. Bootstrap is an execution boundary. PCC is persistent project memory.
Knowledge Artifacts are not Project Docs. Knowledge Artifacts are investigation outputs — potentially ephemeral. Project Docs are curated human-readable explanations.

The intent: frontmatter field

All workflow-produced artifacts should declare their class:

---
intent: system_doc | project_doc | knowledge_artifact | pcc | bootstrap
---

Optional for existing artifacts. Required for all new artifacts produced by v0.4+ workflows.

Workflow to artifact intent mapping

Workflow Artifact intent Rationale
workflow-new-feature project_doc Describes a feature of the user's project
workflow-new-agent system_doc Defines a new Volon agent role
workflow-new-skill system_doc Creates a new Volon skill
workflow-app-investigation knowledge_artifact Investigation output; may be ephemeral
workflow-docs-review system_doc or project_doc Depends on target
pcc-refresh pcc Agent-only context
bootstrap-update bootstrap Execution handoff
Override rule: When Volon developers use workflow-new-feature to build Volon itself, set intent: system_doc to override the default.
