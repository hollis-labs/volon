Configuration

Reference for volon.yaml — all sections and their options.

File resolution order

Volon checks for the config file in this order:

  1. volon.yaml at repository root
  2. .volon/volon.yaml

Unrecognized keys are ignored for forward compatibility.

agents

agents:
  orchestrator:
    enabled: true
    default_role: pm_orchestrator
    single_writer: true       # Enforces Orchestrator-only writes
    read_bootstrap_first: true
  subagents:
    enabled: false            # Enable bounded sub-agents
    max_per_task: 2
    max_per_run: 6
    allow_commands: false
    forbid_writes: true       # Sub-agents are read-only
    no_recursive_spawns: true

storage

storage:
  backend: files          # files | nanite
  files:
    root: ".volon/tasks"  # Task file location (files backend)
  nanite:
    vault: null           # null = nanite's active vault
    tag_prefix: "volon/"
    default_priority: "B" # A | B | C

observability

observability:
  verbosity: normal       # quiet | normal | verbose | trace
  write_run_log: true
  write_decision_log: true
  redact_secrets: true
  log_dir: ".volon/logs"

git

git:
  use_worktrees: true
  worktree_root: ".worktrees"
  branch_prefix: "volon/"
  pr_mode: optional       # off | optional | required
  pr:
    base_branch: auto
    title_prefix: "volon:"
    body_template: ".volon/templates/pr-body.md"
  commit_mode: iteration  # iteration | isolated
  auto_commit: false      # if true, /commit-task runs at finalize
commit_mode: iteration — batch commit per iteration referencing all tasks. isolated — one commit per task: volon: TASK-ID — title.

pcc

pcc:
  enabled: true
  location: ".volon/pcc"
  commit_to_git: true
  refresh:
    on_workflow_start: true
    on_git_change: true
    scheduled: false
  limits:
    max_files: 25
    max_section_words: 400  # Per file, not total

workflows

workflows:

  quality_loop:

  app_investigation:
    enabled: true
    defaults:
      depth: deep              # surface | deep
      output_recommendations: false

Model selection follows a hierarchy: task-level override → SKILL.md tier → agent cap → workflow default → models.default.
