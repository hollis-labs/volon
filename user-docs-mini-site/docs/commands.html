Commands Reference

All user-invoked slash commands with inputs, outputs, and behavior.

Command principles

  • — Commands are safe defaults.
  • — Commands are idempotent where possible.
  • — Commands respect Orchestrator Mode (single writer).
  • — Commands externalize state before any context reset.

Task commands

/task-create "title"

Create a new task file in .volon/tasks/ from a title. Prompts for description, priority, and tags.

/task-update [task-id] [field=value]

Update one or more fields on an existing task. Appends a timestamped entry to the Updates section.

/task-list [status=] [priority=] [tag=]

List tasks matching the given filters. Returns a table of id, title, status, and priority.

/backlog-task "title"

Capture an idea or future item in .volon/backlog/ with status captured. Not yet an active task.

Pause / Resume

/pause-task [mode] [note]

Pause ongoing work by writing state to disk and updating bootstrap with a resume point.

soft — pause and optionally continue in the same session (default)
restart — pause, then start a fresh session and run /resume-task
compact — pause, compact context, then resume
/pause [note]

Alias for /pause-task soft [note].

/resume-task [note]

Resume from .volon/bootstrap.md. Re-grounds from all state files, identifies the paused task, and continues execution.

/resume [note]

Alias for /resume-task [note].

Workflow commands

/workflow-new-feature "name"

Guided 7-stage workflow for new features. Produces ideation, requirements, PRD, spec, plan artifacts and tasks.

/workflow-docs-review

Full PCC-grounded review of all documentation. Updates stale docs against current PCC truth.

/workflow-app-investigation "name" [scope=path] [depth=surface|deep]

Investigation workflow producing 5 Knowledge Artifacts. Does not create development tasks.

/workflow-new-agent "name"

Define a new Volon agent role with purpose, scope, context requirements, and interface spec.

/workflow-new-skill "name" [plugin=dir] [standalone=true|false]

Create a new Volon skill with SKILL.md spec, requirements, and plugin.json registration.

/workflow-new-slash-command "name"

Define a new user slash command with spec and backing skill mapping. Registers in commands docs.

/workflow-new-plugin "name"

Scaffold a new Volon plugin with plugin.json spec and directory structure plan.

Git commands

/commit-task [task-id] [mode=iteration|isolated]

Commit changes for a completed task. Respects git.commit_mode from config. Does not push.

/pr-open [title] [base=branch]

Push current branch and open a pull request. Respects git.pr_mode.

/worktree-start [name]

Create a new git worktree for isolated task execution. Branch prefix set by git.branch_prefix.

System commands

/pcc-refresh [scope]

Update the Project Context Cache. Reads git signals and updates only impacted PCC sections.

/bootstrap-update

Write .volon/bootstrap.md with the current state. Call at the end of each iteration to enable clean restarts.

/quality-run

Scan codebase for dead code, security issues, correctness, and performance smells. Creates tasks per quality.on_issue.default_action.

/docs-sync

Change-driven doc sync. Intersects git changes with doc targets. No-op if no overlap.
