Workflow Contracts

Execution contracts for all Forge workflow types.

Design goals

  • — Composable workflows: interactive, one-shot, looped, or scheduled.
  • — Each step produces predictable artifacts with frontmatter.
  • — Handoffs pass minimal context; sessions can be reset between iterations.
  • — Default execution role is Orchestrator (single writer).

Artifact conventions

All workflow artifacts are Markdown files with YAML frontmatter:

---
id: feat-YYYY-MM-DD-slug
type: idea | requirements | prd | spec | plan
intent: system_doc | project_doc | knowledge_artifact | pcc | bootstrap
status: draft
project: "repo-name"
tags: ["feature"]
priority: B
created_at: YYYY-MM-DD
updated_at: YYYY-MM-DD
---

Include sections: Summary, Decisions, Open questions, Evidence (if derived from repo).

Core workflow: new-feature

Artifact intent: project_doc (default). Override to system_doc when building Forge itself.

1
Preflight — /pcc-refresh
2
Ideation — artifacts/ideas/<slug>.md
3
Requirements — artifacts/requirements/<slug>.md
4
PRD — artifacts/prd/<slug>.md
5
Spec — artifacts/spec/<slug>.md
6
Plan — artifacts/plan/<slug>.md
7
Create/triage tasks
8
Finalize iteration — /bootstrap-update

Core workflow: docs-review

Full PCC-grounded scan. Always checks all target files against PCC ground truth. Produces updates to system_doc or project_doc artifacts.

  1. Preflight: /pcc-refresh
  2. Update target artifacts (minimal diffs against PCC)
  3. Sync PCC
  4. Optional PR
  5. Finalize: /bootstrap-update

Command: /workflow-docs-review

Extension workflows

Guided workflows for authoring new Forge extensions. Each follows the staged pattern: preflight → ideation → requirements → spec → plan → tasks → finalize.

Command Output path
/workflow-new-agent "name" artifacts/agents/<slug>/
/workflow-new-skill "name" artifacts/skills/<slug>/
/workflow-new-slash-command "name" artifacts/slash-commands/<slug>/
/workflow-new-plugin "name" artifacts/plugins/<slug>/

Investigation workflow

Investigates applications and codebase components without development intent. Produces Knowledge Artifacts. 

/workflow-app-investigation "name" [scope=path] [depth=surface|deep]
scope.mdInvestigation boundaries and questions (Stage 2)
discovery.mdFile inventory, dependencies, entry points (Stage 3)
analysis.mdComponent breakdown, data flows, patterns, risks (Stage 4)
findings.mdSynthesized insights with confidence levels (Stage 5)
report.mdShareable executive summary, status: complete (Stage 6)
Key distinction: Investigation does NOT produce tasks or development artifacts. Idempotent — re-run safely, skips existing stages.

Looping behavior (task-driven)

  1. Read .forge/bootstrap.md if present.
  2. Drive execution from .forge/tasks/.
  3. Update task states and write run logs.
  4. Finalize iteration: /bootstrap-update
  5. Idempotent: if no action, output DONE.
