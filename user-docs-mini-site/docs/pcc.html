Project Context Cache

Compact, truth-checked agent context that eliminates repeated repository scanning.

Purpose

PCC is an agent-only set of short, structured files that reduce the need to re-scan the repository on every session. Each file covers a specific domain (project overview, architecture, conventions, workflows, backlog, decisions) and is kept under a configurable word limit.

PCC is not user documentation. It is a compact context pack for agents. Do not update PCC and call it "updating the docs."

Required files

00_project.md What this repo is, goals/non-goals, repo type signals
01_architecture.md Components, boundaries, data flow
02_conventions.md Coding standards, branching, naming, test commands
03_workflows.md Run/test/release commands, environments
04_backlog.md Current sprint pointer and top roadmap items
05_decisions.md Lightweight ADR log (append-only)
Optional files
06_interfaces.mdPublic APIs and CLIs
07_security.mdAuth surfaces, secret handling, threat notes
08_observability.mdLogging, metrics, and tracing notes

Style rules

  • — Keep each section short (configured by max_section_words, default 400).
  • — Prefer bullets and links over prose.
  • — Never invent behavior; mark unknowns as TBD.
  • — Always include an Evidence section (files/commands used to derive the content).

Refresh protocol

Run /pcc-refresh to update the cache. Only impacted sections are updated.

1 Read existing PCC files.
2 Check git signals: git status, git diff --name-only, recent log window.
3 Update only PCC sections impacted by changed files.
4 Write run log and decision log (if observability is enabled).
PCC refresh triggers: on_workflow_start, on_git_change. Configurable in forge.yaml under pcc.refresh.
