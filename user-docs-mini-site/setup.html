Setup

Get Forge running in your repository.

Prerequisites

1
Claude Code

Forge is designed to run with Claude Code (Anthropic's CLI). Ensure you have Claude Code installed and authenticated with an API key.

2
Git repository

Forge operates inside a Git repository. Initialize one with git init or use an existing repo.

3
Forge plugin files

The Forge plugin directory (plugins/) and configuration files (forge.yaml, .forge/) must exist in the repository root.

Repository structure

Forge uses the following directories at your repository root: 

your-repo/
├── forge.yaml              # System configuration
├── .forge/
│   ├── agent-boot.md       # Agent startup instructions
│   ├── bootstrap.md        # Current iteration state
│   ├── tasks/              # Task files (TASK-YYYYMMDD-NNN.md)
│   ├── backlog/            # Captured backlog items
│   ├── logs/               # Run and decision logs
│   └── pcc/                # Project Context Cache (agent-only)
├── plugins/
│   └── workflows/          # Workflow skill definitions
└── docs/                   # System documentation

Minimal forge.yaml

Create forge.yaml at the repository root with at minimum: 

version: 1

agents:
  orchestrator:
    enabled: true
    single_writer: true
    read_bootstrap_first: true
  subagents:
    enabled: false

storage:
  backend: files
  files:
    root: ".forge/tasks"

observability:
  write_run_log: true
  log_dir: ".forge/logs"

git:
  use_worktrees: false
  commit_mode: iteration
  auto_commit: false

pcc:
  enabled: true
  location: ".forge/pcc"

models:
  default: claude-sonnet-4-6

See the Configuration reference for all available options.

First run walkthrough

1
Open Claude Code in your repository

Start a Claude Code session at the repository root. Forge reads the agent-boot file automatically when configured as a project custom instruction.

2
Initialize the Project Context Cache
/pcc-refresh

This scans the repository and populates .forge/pcc/ with compact context files for subsequent runs.

3
Create your first task or start a workflow
# Create a task
/task-create "Add user authentication"

# Or start a feature workflow
/workflow-new-feature "user-auth"
4
Finalize the iteration
/bootstrap-update

This writes .forge/bootstrap.md with the current state, enabling clean restarts in future sessions.

Forge is set up

Future sessions start by reading .forge/bootstrap.md to determine what to do next. No state is held in memory.
