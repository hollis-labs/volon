Use Cases

Common scenarios and how to apply Volon to them.

Adding a new feature

Feature development workflow

Context

You need to add a well-defined feature to your project and want staged artifacts (ideas, requirements, PRD, spec, plan) before writing code.

Invoke
/workflow-new-feature "user-auth"
Expected outcome

Staged artifacts in artifacts/, tasks created in .volon/tasks/, bootstrap updated with next actions.

Investigating an unfamiliar codebase

App investigation workflow

Context

You've inherited a codebase or component and need to understand its architecture, dependencies, and risks before making changes.

Invoke
/workflow-app-investigation "frontend" scope=src/
Expected outcome

Knowledge Artifacts in artifacts/knowledge/frontend/: scope, discovery, analysis, findings, and executive report.

Creating a new Volon skill

Extension workflow

Context

You want to add a reusable capability to Volon — a skill that agents can invoke, with a defined SKILL.md contract and optional plugin registration.

Invoke
/workflow-new-skill "dependency-audit"
Expected outcome

Artifacts in artifacts/skills/dependency-audit/, tasks for SKILL.md creation and plugin.json registration.

Pausing work and resuming cleanly

Pause/resume protocol

Context

You're mid-task and need to end the session. You want to resume later in a fresh session without losing context or having to re-explain the situation.

Invoke
# To pause:
/pause-task restart "mid-task: DB schema needs review"

# In the next session:
/resume-task
Expected outcome

Task status set to paused, bootstrap updated with resume point. New session reads bootstrap and continues from where work stopped.

Running a code quality pass

Quality workflow

Context

You want to scan the codebase for dead code, security issues, correctness problems, or performance smells and get actionable task items.

Invoke
/quality-run
Expected outcome

Scan results logged, issues triaged. New tasks created per the quality.on_issue.default_action config setting.

Capturing and promoting backlog items

Backlog workflow

Context

You have ideas or future work you want to track without committing to immediate execution. Later, you promote the item when it's time to act.

Invoke
# Capture an idea to backlog
/backlog-task "Add export to CSV feature"

# Later: promote to active task
/task-create --from-backlog BACKLOG-001
Expected outcome

Backlog item created in .volon/backlog/ with status captured. On promotion, status becomes promoted and an active task is created.
