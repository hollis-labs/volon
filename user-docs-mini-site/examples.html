Feature workflow

/workflow-new-feature "user-auth"

Stage 1
Preflight — PCC refresh

Reads volon.yaml and .volon/bootstrap.md. Updates .volon/pcc/ if stale.

PCC current — 6 files, all under 400-word cap
Stage 2
Ideation

Generates initial ideas for the feature based on PCC context.

artifacts/ideas/user-auth.md created
  intent: project_doc
  status: draft
Stage 3
Requirements

Expands ideas into structured requirements with acceptance criteria.

artifacts/requirements/user-auth.md created
Stage 4
PRD + Spec + Plan

Produces product requirements document, technical spec, and implementation plan.

artifacts/prd/user-auth.md created
artifacts/spec/user-auth.md created
artifacts/plan/user-auth.md created
Stage 5
Task creation + Bootstrap update

Creates implementation tasks and finalizes the iteration.

.volon/tasks/TASK-20260222-002.md  (todo)
.volon/tasks/TASK-20260222-003.md  (todo)
.volon/tasks/TASK-20260222-004.md  (todo)
.volon/bootstrap.md updated — 3 new todos
Investigation workflow

/workflow-app-investigation "frontend"

Produces 5 Knowledge Artifacts. Does not create development tasks.

scope.md Investigation boundaries: what files to examine, what questions to answer, what depth setting is active.
discovery.md File inventory, entry points, dependency graph, external API calls detected.
analysis.md Component breakdown, data flows, architectural patterns, identified risks and coupling.
findings.md Synthesized insights with confidence levels (high/medium/low). Key decisions surface.
report.md Shareable executive summary. Status set to complete.
Idempotent: Re-running the command skips any stage whose artifact already exists. Safe to restart after interruption.
Pause/Resume

/pause-task restart "DB schema review needed"

Session A (current session)
# You are mid-task on TASK-20260222-005 when you realize
# you need to stop and consult on the DB schema.

/pause-task restart "mid-task: DB schema review needed before continuing"

# Volon writes:
.volon/tasks/TASK-20260222-005.md
  status: paused
  Updates: 2026-02-22 — paused: DB schema review needed

.volon/bootstrap.md
  paused_task_id: TASK-20260222-005
  resume_hint: "DB schema review needed; resume when resolved"

# Volon prints:
Resume instructions:
  1. Start a new session in the repo root.
  2. Run: /resume-task
     (or pass a note: /resume-task "schema reviewed, proceed")
Session B (next session)
/resume-task "schema reviewed — use users_v2 table"

# Volon:
# 1. Reads .volon/bootstrap.md → finds paused_task_id
# 2. Reads TASK-20260222-005.md → status: paused
# 3. Resumes execution from last known state
# 4. Applies the user note: use users_v2 table
# 5. Continues the loop normally
