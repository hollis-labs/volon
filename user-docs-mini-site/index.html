Multi-session orchestration for repository automation

Forge coordinates human-directed workflows, task execution, and knowledge artifact generation through a single-writer Orchestrator that delegates bounded work to sub-agents.

Get Started Read the Docs

Core capabilities

Orchestrated Workflows

Composable, staged workflows for new features, code review, investigations, and Forge extensions. Each stage produces verifiable artifacts.

Project Context Cache

Compact, truth-checked context files in .forge/pcc/ eliminate repeated repository scanning and keep agents grounded.

Task Model

Structured Markdown task files with full lifecycle tracking: todo, doing, paused, blocked, done.

Backlog and Promotion

Capture ideas in .forge/backlog/. Promote items to active tasks when ready. Clear separation between captured and in-flight work.

Pause and Resume

Deterministic state externalization via disk, not memory. Pause mid-task and resume cleanly in a fresh session. Works across context resets.

Artifact Intent

Five artifact classes — System Docs, Project Docs, Knowledge Artifacts, PCC, and Bootstrap — each with explicit intent: frontmatter.
