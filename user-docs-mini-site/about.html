About Forge

Philosophy, design principles, and what Forge is built to do.

What is Forge?

Forge is a multi-session orchestration system for repository automation. It coordinates human-directed workflows, task execution, and knowledge artifact generation through a single-writer Orchestrator that delegates bounded, read-only work to sub-agents.

Forge runs inside your repository. It uses plain Markdown files for tasks, logs, and context — no databases, no external services required. State lives on disk. Sessions can be started, paused, and resumed deterministically.

Philosophy

Project-agnostic

Forge does not assume a specific language, framework, or deployment model. It adapts to any repository through configuration and PCC.

Composable

Workflows, skills, and plugins are independent units. New capabilities can be added without modifying the core system.

State on disk, not in memory

Every significant state transition writes to a file. Sessions can be ended and restarted at any time without losing progress. Chat context is not a source of truth.

Design principles

Principle Description
Single writer Only the Orchestrator may modify tasks, logs, PCC, or bootstrap. Sub-agents are read-only.
Idempotent Workflows are safe to re-run. Stages that already have artifacts are skipped.
Minimal diffs Each task produces small, verifiable changes. Large tasks are split into bounded increments.
Artifact intent Every workflow artifact declares its class via intent: frontmatter. Five classes are defined.
Ground truth in files Agents always re-ground from repository artifacts. Conversation context is not trusted across sessions.
Bounded delegation Sub-agents execute a single scoped objective. No recursive spawning. Orchestrator integrates results.

v0.5 milestone

Forge v0.5 completes six foundational epics, establishing the full core system.

EPIC-001 — Artifact Classes
Five-class artifact taxonomy with intent frontmatter and workflow domain separation.
EPIC-002 — Workflow Lifecycle Suite
Full workflow create/edit/clone/deprecate commands and extension workflows for agents, skills, plugins, and slash commands.
EPIC-003 — Investigation Workflows
Application investigation workflow producing Knowledge Artifacts (scope, discovery, analysis, findings, report).
EPIC-004 — Orchestrator Mode
Single-writer Orchestrator role with bounded sub-agent delegation, model selection, and loop protocol.
EPIC-005 — Pause/Resume Protocol
Deterministic pause and resume via disk-state externalization. Works across context resets and session boundaries.
EPIC-006 — Model Configuration
Hierarchical model selection by workflow stage, task override, agent cap, and context size policy.

Non-goals

  • — Forge is not a code editor, IDE plugin, or autonomous agent without human direction.
  • — Forge does not replace version control — it uses Git as infrastructure.
  • — Forge does not manage deployment, CI/CD pipelines, or production infrastructure directly.
  • — Forge does not require a server, database, or external API beyond the AI model provider.
  • — Forge does not auto-merge pull requests or push without human confirmation.
